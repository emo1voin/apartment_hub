## 🗄️ Структура базы данных

### Таблицы
- **users** - пользователи системы
- **hotels** - дома/отели (с полями модерации)
- **rooms** - квартиры (с полями модерации)
- **bookings** - бронирования
- **reviews** - отзывы
- **amenities** - удобства
- **hotel_amenities** - связь между домами и удобствами
- **room_amenities** - связь между квартирами и удобствами

### Поля модерации
```
hotels:
  - owner_id (владелец)
  - status (pending/approved/rejected)
  - rejection_reason (причина отклонения)
  - approved_at (дата одобрения)
  - approved_by (кто одобрил)

rooms:
  - status (pending/approved/rejected)
  - rejection_reason (причина отклонения)
  - approved_at (дата одобрения)
  - approved_by (кто одобрил)
```

## 🔧 Установка и запуск

### Требования
- PHP 8.2+
- MySQL 8.0+
- Composer
- Node.js (для Tailwind CSS)

### Установка
```bash
# Клонирование репозитория
git clone <repo-url>
cd example-app

# Установка зависимостей
composer install

# Копирование .env файла
cp .env.example .env

# Генерация ключа приложения
php artisan key:generate

# Миграция базы данных
php artisan migrate

# Заполнение тестовыми данными
php artisan db:seed

# Запуск сервера
php artisan serve
```

### Тестовые аккаунты
- **Админ:** admin@booking.com / password
- **Пользователь:** user@booking.com / password

## 🚀 Маршруты

### Публичные маршруты
- `GET /` - главная страница
- `GET /hotels` - список домов
- `GET /hotels/{id}` - детали дома
- `GET /rooms/{id}` - детали квартиры

### Маршруты пользователя (требуют авторизации)
- `GET /bookings` - мои бронирования
- `POST /bookings` - создать бронирование
- `GET /bookings/{id}` - детали бронирования
- `POST /bookings/{id}/cancel` - отменить бронирование
- `GET /my/hotels` - мои дома
- `POST /my/hotels` - добавить дом
- `GET /my/hotels/{id}/edit` - редактировать дом
- `PUT /my/hotels/{id}` - сохранить изменения дома
- `DELETE /my/hotels/{id}` - удалить дом
- `GET /my/rooms` - мои квартиры
- `POST /my/rooms` - добавить квартиру
- `GET /my/rooms/{id}/edit` - редактировать квартиру
- `PUT /my/rooms/{id}` - сохранить изменения квартиры
- `DELETE /my/rooms/{id}` - удалить квартиру

### Маршруты админа (требуют роли admin)
- `GET /admin/moderation` - панель модерации
- `POST /admin/moderation/hotels/{id}/approve` - одобрить дом
- `POST /admin/moderation/hotels/{id}/reject` - отклонить дом
- `POST /admin/moderation/rooms/{id}/approve` - одобрить квартиру
- `POST /admin/moderation/rooms/{id}/reject` - отклонить квартиру

### API маршруты
- `POST /api/v1/auth/register` - регистрация
- `POST /api/v1/auth/login` - вход
- `GET /api/v1/hotels` - список домов
- `GET /api/v1/rooms` - список квартир
- `POST /api/v1/bookings` - создать бронирование (требует токен)

## 🐛 Исправленные проблемы

### Проблема 1: Конфликт маршрутов
**Проблема:** Два маршрута с одинаковым именем `bookings.store`
**Решение:** Переименованы API маршруты на `api.bookings.store`

### Проблема 2: Ошибка базы данных
**Проблема:** Модель использовала `check_in_date`, но в БД колонка `check_in`
**Решение:** Обновлены имена полей в модели и контроллере

### Проблема 3: Форма не отправляется
**Проблема:** MCP browser logger перехватывал запросы
**Решение:** Отключен browser logger в `.env`

### Проблема 4: Ошибка модерации
**Проблема:** Попытка доступа к свойству null (owner)
**Решение:** Добавлена проверка на наличие владельца в view
